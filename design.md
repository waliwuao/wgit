# wgit(git辅助工具)

wgit是一个面向git初学者的规范化git工具，可以通过模糊搜索和选择的形式减少git命令的记忆量，并且提高git使用的规范性。
使用这个工具默认使用着已经安装了git

## 命令功能描述

### init

在工作目录下运行wgit init后会先检查这个目录是不是一个git仓库，如果不是，运行git init,设置主分支为main,并添加限制，禁止用户在主分支下修改和提交。wgit的配置文件存在.git/中，避免污染仓库

### add

执行add后进入选择交互界面（选择交互界面概述如下，第一行为搜索行可以模糊搜索并及时更新下面的选项，按空格键选择文件，回车键确认提交，上下方向键移动选项，左方向键取消全选，右方向键全选。确定后通过git add提交所有选择的文件

### commit

commit会先检查是否有需要提交的内容，如果没有就提示并退出，如果有需要提交的内容就进入选择交互界面选择提交的类型，如feat,doc（与add不同，add是多选，commit是单选，按回车直接选择），选择完成后约束编写提交内容进入一个文本编辑器，类似vim,可以填写scope,subject,body,按esc确认保存并提交。

### start

在当前分支的基础上创建特定分支，要求用户先选择分支类型，选择完成后填写分支名称，回车确认后创建分支

### finish

finish命令只可以在非主分支运行，会将当前分支合并到父分支，会自动squash并进入和commit里一样的文本编辑器，约束性填写合并信息，确认后合并。注意，合并时如果发生了conflict会触发以下选项，回退：撤销此次合并，继续：告诉用户自行解决冲突，并且告诉用户在解决冲突后应该运行什么命令。如果是合并到主分支，会要求打tag,tag是限制性的，会先展示上次的tag是什么，然后要求用户按照标准填写新tag

### remote

可以自动检测远程仓库，可以添加远程仓库并设置别名

### switch

切换分支，选择式界面

### undo

支持不同类型和不同级别的回退，先选择是基于提交的回退还是基于操作的回退，再选择是什么级别的回退hard,soft

### sync

自动pull和push,会自动stash流程，如果出现冲突则提供和finish一样的选项

### update

由于这个工具会发布在github上提供下载，并且这个工具经常会更新，update命令会自动检测是否有新版本发布并在确认后自动安装

### menu

进入选择交互界面，与commit的单选式大致相同，可以选择需要执行的命令，还包含一个exit支持退出，当wgit不接参数运行时，默认执行menu

## 要求

这个工具包含大量的选择，选择的界面设计要美观，实际上包含单选和多选两种类型，可以分别参照commit和add.

每次运行git命令时都要打印并通过颜色与其它命令进行区分

TUI设置要美观，可以使用rust的TUI库来简化开发并提高效果，可以按照需求对文件结构进行一些改良

所有的命令、选项和输出都必须是英文
